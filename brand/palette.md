# InstantLink Brand

The mark is a **flat instant-photo frame** (true Instax portrait proportion,
thick bottom lip) with a **camera aperture** cut into the photo window — the
subject rendered literally, not an abstract "connection" glyph. Flat color, no
gradient. Direction and rationale: `docs/research/2026-design-direction.md`
(Colorforms / Anti-AI-Crafting; drawn from real instant film).

## Assets

| File | Use |
|------|-----|
| `instantlink-icon.svg` | Full-bleed app icon (1024²) — ink ground, bone frame, amber aperture. iOS masks the corners. |
| `instantlink-mark.svg` | Transparent mark for **light** surfaces (ink frame). |
| `instantlink-mark-dark.svg` | Transparent mark for **dark** surfaces (bone frame). |
| `instantlink-logo.svg` | Horizontal symbol + wordmark lockup. Outline the text to paths before external distribution. |

Rasterize with `cairosvg`. The iOS app icon and the Bridge boot splash are
generated by `brand/render-assets.sh`.

## Palette — flat, drawn from real instant film

| Token | Hex | Use |
|-------|-----|-----|
| Safelight | `#E0552B` | The single brand accent — a darkroom-safelight amber-red. Aperture, links, primary action. Flat, never a gradient. |
| Ink | `#1A1613` | Warm photographic near-black — grounds, body text, the frame on light. |
| Window | `#14110E` | The dark "photo" area inside the frame. |
| Bone | `#F1EADA` | Warm instant-film ivory — the frame, light surfaces. Not a cold paper white. |

Neutrals for UI are warm greys biased toward the ink, never a pure mid-grey.

## Type

The platform system sans, set with intent — **SF Pro** on Apple platforms,
**DejaVu Sans / Noto** on the Bridge LCD. The wordmark is one weight (~650) with
tight tracking; the symbol carries the color and character, so the wordmark stays
quiet and ink-only. No display webfont is imposed.

## Usage notes

- The mark is subject-literal: the frame is a real instant photo, the aperture a
  real lens. Never replace it with an abstract node/network glyph.
- Flat only. No gradients, no glossy specular — those read as AI-generated (see
  the research doc).
- **Safelight amber-red is a brand accent, not a status color.** On the Bridge
  LCD's status vocabulary, keep semantic green (ready) / amber-yellow (warning) /
  red (error) distinct; the brand accent applies to non-status chrome (selection,
  sliders, primary actions) and the boot splash.
- Minimum mark size ~16 px; the aperture blurs to a warm dot but the framed-photo
  silhouette still reads. Don't crop the bottom lip.
